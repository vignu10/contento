# PRD: Content Editing Capabilities

## Problem
Users cannot edit their generated content or delete content they no longer need. This limits the usefulness of the platform—users want to customize outputs before posting and manage their content library.

## Users
- Content creators who want to tweak AI-generated posts
- Marketers who need to adjust tone or add specific details
- Users who uploaded wrong content and want to delete it

## User Stories
- As a user, I want to edit the AI-generated Twitter thread to fix errors or add my voice
- As a user, I want to edit the LinkedIn post to adjust the tone or add hashtags
- As a user, I want to delete content I uploaded by mistake
- As a user, I want to see which outputs I've edited vs. original AI versions

## Proposed Solution
Add editing and deletion capabilities for content and outputs:

**Output Editing**:
- Each output card has an "Edit" button
- Clicking opens a modal or inline edit mode
- Textarea with the current output content
- "Save" and "Cancel" buttons
- Original vs. edited version tracking in `editedData` field
- "Revert to original" option

**Content Deletion**:
- Add "Delete" button on content cards in dashboard
- Add "Delete" button on content detail page
- Confirmation dialog ("Are you sure? This cannot be undone")
- Cascade delete: deleting content also deletes all associated outputs

**UI Updates**:
- Edit button on each output card (next to Copy button)
- Delete button on content cards (three-dot menu or trash icon)
- Toast notifications for successful edits/deletions
- Visual indicator for edited outputs (e.g., "Edited" badge)

## Scope
### In Scope
- Edit outputs (Twitter, LinkedIn, Newsletter, SEO, Instagram)
- Edit TikTok clip scripts and hooks
- Delete content with cascade to outputs
- Inline or modal editing interface
- Visual feedback for edited items
- Confirmation dialogs for destructive actions

### Out of Scope
- Edit transcript (separate feature)
- Edit content title/source URL (create new content instead)
- Bulk editing (edit multiple outputs at once)
- Undo/redo functionality
- Version history for edits

## Technical Approach
- **Frontend**: Add Edit button to output cards, modal component for editing
- **API**: PUT endpoint already exists at `/api/content/[id]/outputs/route.ts`
- **Frontend**: Add Delete button to content cards, DELETE endpoint exists
- **DB**: `editedData` field already exists on Output model
- **State**: Optimistic UI updates for better UX
- **Testing**: Verify edit saves correctly, delete cascade works

## Acceptance Criteria
- [ ] Each output card has an Edit button
- [ ] Clicking Edit opens edit modal with current content
- [ ] Saving edits updates the `editedData` field in database
- [ ] Edited outputs display the edited version (not original)
- [ ] "Revert to original" restores the AI-generated version
- [ ] Content cards have a Delete option
- [ ] Delete action shows confirmation dialog
- [ ] Deleting content removes all associated outputs
- [ ] Toast notifications appear for successful edits/deletions
- [ ] Edited outputs have visual indicator (badge or icon)
- [ ] Tests verify edit and delete functionality

## Design Notes
- Use a modal for editing to keep context
- Auto-save drafts (optional, future enhancement)
- Show character count for Twitter (280 chars)
- For TikTok clips, allow editing hook and script separately
- Delete should be permanent (no trash/restore for MVP)
