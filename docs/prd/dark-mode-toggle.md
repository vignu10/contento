# PRD: Dark Mode Toggle

## Problem
The app has dark mode CSS classes (using Tailwind dark: prefix) but no way for users to switch between light and dark themes. Users have personal preferences for theme, and many expect a dark mode toggle in 2026.

## Users
- Users who prefer dark mode to reduce eye strain
- Users working in low-light environments
- Users who switch between light/dark based on time of day
- All users who expect theme control

## User Stories
- As a user, I want to toggle between light and dark mode
- As a user, I want my theme preference to be remembered
- As a user, I want the theme to match my system preference by default

## Proposed Solution
Add dark mode toggle to the app:

**Toggle Control:**
- Toggle button in header (sun/moon icon)
- Positioned near user info/logout
- Clear visual feedback (sun for light, moon for dark)
- Smooth transition between themes

**Theme Persistence:**
- Save preference in localStorage
- Check for saved preference on page load
- Fall back to system preference if no saved preference
- Apply theme class to <html> or <body>

**Theme Detection:**
- Check `window.matchMedia('(prefers-color-scheme: dark)')` for system preference
- Default to system preference on first visit

**Implementation Options:**
1. **Client-side only** (simplest): Use localStorage, apply class in useEffect
2. **Server-side with cookies** (better SEO): Set cookie, read in middleware

For MVP, use client-side only.

## Scope
### In Scope
- Dark/light mode toggle button in header
- Theme preference saved in localStorage
- System preference detection as default
- Smooth CSS transitions for theme switch
- Toggle icon updates based on current theme

### Out of Scope
- Theme picker with multiple color schemes
- Custom accent colors
- Per-page theme settings
- Theme preference sync across devices

## Technical Approach
- **Frontend**: Toggle component in header
- **State**: useTheme hook with localStorage
- **Tailwind**: Use `dark:` prefix for all dark mode styles (already in place)
- **Transitions**: Add transition-all to body for smooth theme switch
- **Next.js**: use client directive for theme provider component
- **Testing**: Verify theme persists, toggle works, system preference detection
