# PRD: Real AI Content Generation

## Problem
Currently, the content repurposing pipeline generates mock outputs (fake Twitter threads, LinkedIn posts, etc.). Users expect actual AI-generated content that is relevant to their specific content, not generic placeholders.

## Users
- Content creators who need social media posts from their content
- Marketers who need multiple formats from one piece of content
- Agencies who repurpose client content

## User Stories
- As a user, I want the AI to generate Twitter threads that reference specific points from my content
- As a user, I want LinkedIn posts that include relevant hashtags and professional tone
- As a user, I want newsletter drafts that summarize my key insights
- As a user, I want TikTok clip suggestions with timestamps and hooks from my actual content

## Proposed Solution
Implement real AI content generation using OpenAI GPT-4 or GPT-4o:

**Input**: Transcript + metadata (title, sourceType, etc.)

**Output Generation** (via structured prompting):
1. **Twitter Thread**: Hook + 5-8 tweets, each quoting or referencing transcript
2. **LinkedIn Post**: Professional summary + insights + hashtags
3. **Newsletter**: Structured summary with key takeaways and action items
4. **TikTok Clips**: 3-5 hooks with timestamps, scripts for 30-60s clips
5. **Quote Graphics**: 5-10 memorable quotes extracted from transcript
6. **SEO Summary**: Blog post summary with keywords
7. **Instagram Caption**: Engaging caption + relevant hashtags

**AI Service**:
- New service `src/services/openai.ts`
- Use GPT-4o for cost-effective high-quality outputs
- System prompts for each format type
- Extract structured JSON responses
- Handle rate limits with queue/backoff

## Scope
### In Scope
- GPT-4o integration for all 7 output formats
- Structured prompts for each format type
- JSON response parsing and validation
- Error handling for API failures
- Rate limiting and retry logic
- Transcript-based content generation (not generic)

### Out of Scope
- Custom voice/style training (separate feature)
- Output templates/brand kits (separate feature)
- User feedback loop to improve outputs
- Alternative AI providers (Claude, etc.)

## Technical Approach
- **Backend**: New service `src/services/openai.ts` with format-specific functions
- **API**: Update `src/app/api/content/[id]/generate/route.ts` to use real AI
- **Queue**: Bull job for async generation (after transcription)
- **Prompts**: System prompts in code or config for each format
- **Structured Output**: Use GPT-4o's JSON mode for reliable parsing
- **Cost Tracking**: Log API usage for billing limits
- **Testing**: Mock OpenAI in tests, verify prompt quality

## Acceptance Criteria
- [ ] All 7 output formats generate real, transcript-specific content
- [ ] Twitter threads reference actual points from transcript
- [ ] TikTok clips include accurate timestamps from content length
- [ ] Quote graphics extract actual quotes from transcript
- [ ] Generated content is grammatically correct and coherent
- [ ] API errors are handled gracefully with retry (3 attempts)
- [ ] Rate limits are respected (don't exceed OpenAI quotas)
- [ ] Output generation is triggered after transcription completes
- [ ] Tests verify output structure and quality with mocked API
- [ ] Cost per content piece is tracked for billing

- Prompt engineering is critical for quality
- Use temperature 0.7 for creativity, 0.3 for structured outputs
