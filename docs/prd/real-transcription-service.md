# PRD: Real Transcription Service

## Problem
Currently, the content repurposing pipeline generates mock transcripts and mock outputs. The core value proposition—transforming real content into multiple formats—cannot work without actual transcription of audio/video content.

## Users
- YouTubers who paste video URLs
- Podcasters who upload audio files
- Coaches/consultants who upload webinars
- Anyone uploading video files (MP4, MOV)

## User Stories
- As a user, I want to upload my YouTube video and get an accurate transcription so that I can repurpose the content
- As a user, I want the transcription to include speaker identification and timestamps for better context
- As a user, I want to see the transcription progress so I know when my content will be ready

## Proposed Solution
Implement real transcription using OpenAI's Whisper API for all audio/video content:

1. **YouTube Videos**: Download audio track, transcribe with Whisper
2. **Audio Uploads**: Direct transcription with Whisper
3. **Video Uploads**: Extract audio, transcribe with Whisper
4. **PDF/Blog**: Skip transcription (text already available)

**Transcription Process**:
- User submits content (URL or file)
- Queue job in Bull for async processing
- Download/extract audio as needed
- Send to Whisper API
- Save transcript to database
- Trigger output generation job
- Update content status to "completed"

**Progress Tracking**:
- Show "Processing..." state in UI
- Auto-refresh content detail page
- Display transcript in collapsible section

## Scope
### In Scope
- OpenAI Whisper integration for transcription
- Audio extraction from video files (FFmpeg)
- YouTube audio download
- Progress tracking via status field
- Error handling for failed transcriptions
- Basic transcript formatting (paragraphs, timestamps)

### Out of Scope
- Speaker diarization (multiple speakers)
- Custom vocabulary/brand terms
- Transcription editing interface
- Transcript export (separate feature)
- Alternative transcription providers

## Technical Approach
- **Backend**: New service `src/services/whisper.ts`
- **API**: Update `src/app/api/content/[id]/generate/route.ts` to call real Whisper
- **Queue**: Bull job for async processing
- **FFmpeg**: Use `@ffmpeg/ffmpeg` for audio extraction from video
- **YouTube**: Use yt-dlp or similar to download audio track
- **DB**: Store transcript in `Content.transcript` field
- **Testing**: Mock Whisper in tests, verify integration with real API

## Acceptance Criteria
- [ ] YouTube URL submission downloads audio and transcribes
- [ ] Audio file uploads (MP3, WAV, M4A) transcribe successfully
- [ ] Video file uploads (MP4, MOV) extract audio and transcribe
- [ ] Transcription status updates from "pending" → "processing" → "completed" or "failed"
- [ ] Transcript is stored in database and displayed in content detail page
- [ ] Transcription fails gracefully with error message on API/network errors
- [ ] Progress is visible to user (processing badge, auto-refresh)
- [ ] Queue handles retries for transient failures (3 attempts)
- [ ] Tests verify transcription flow with mocked Whisper API

## Design Notes
- Processing time: 1-5 minutes for 1-hour content (Whisper is fast)
- Show estimated wait time based on content length
- Allow users to retry failed transcriptions
- Cost: ~$0.006 per minute of audio via Whisper API
