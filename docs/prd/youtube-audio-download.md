# PRD: YouTube Audio Download

## Problem
Users can paste YouTube URLs and the app fetches video metadata, but it cannot download the audio track for transcription. Without the actual audio, the core value proposition (transcribing content) doesn't work for YouTube videos.

## Users
- YouTubers who want to repurpose their own videos
- Content creators who want to extract value from existing videos
- Anyone using YouTube as their content source

## User Stories
- As a user, I want to paste a YouTube URL and have the audio automatically downloaded
- As a user, I want the download to be fast (seconds, not minutes)
- As a user, I want the download to fail gracefully if the video is unavailable

## Proposed Solution
Implement YouTube audio download using yt-dlp or a similar library:

**Download Process**:
1. User submits YouTube URL
2. Validate URL and extract video ID (already implemented)
3. Download audio track in best quality (prefer mp3/wav)
4. Save to local filesystem or S3 (using existing storage service)
5. Store file reference in Content.sourceFile
6. Trigger transcription queue job

**Audio Quality**:
- Download best available audio
- Convert to MP3 for Whisper compatibility
- Target: 128kbps or higher for good transcription quality

**Error Handling**:
- Video not available (private, deleted, region-locked)
- Network timeout
- Invalid video ID
- Rate limiting (add delay between downloads)

## Scope
### In Scope
- YouTube audio download using yt-dlp or youtube-dl-exec
- Audio format conversion to MP3
- File storage (local or S3 based on config)
- Error handling for unavailable videos
- Integration with existing Content creation flow
- Progress tracking during download

### Out of Scope
- Video download (only audio needed for transcription)
- Subtitle/caption extraction (use Whisper instead)
- Multi-language audio tracks
- Playlist support
- Download queue management (use existing Bull queue)

## Technical Approach
- **Backend**: New service `src/services/youtube-download.ts`
- **Library**: `youtube-dl-exec` or `yt-dlp` via Node wrapper
- **Conversion**: FFmpeg (already in dependencies) for audio extraction
- **Storage**: Use existing `uploadFile` from `src/lib/storage.ts`
- **Integration**: Update `src/app/api/content/route.ts` to download audio after video info fetch
- **Queue**: Add download job to Bull queue for async processing

## Acceptance Criteria
- [ ] YouTube URL triggers audio download
- [ ] Audio file is saved and stored in Content.sourceFile
- [ ] Download completes within 30 seconds for typical videos
- [ ] Unavailable videos show clear error message
- [ ] Downloaded audio is in MP3 format
- [ ] File size is reasonable (< 100MB for 1-hour video)
- [ ] Transcription is triggered after successful download
- [ ] Failed downloads show retry option
- [ ] Tests verify download flow with mocked yt-dlp

## Design Notes
- yt-dlp is more reliable than youtube-dl (actively maintained)
- Show download progress in UI (if possible via WebSocket/polling)
- Cache downloads temporarily to avoid re-downloading same video
- Consider using YouTube's caption API as fallback (though Whisper is better)
