# PRD: User Settings & Profile

## Problem
Users have no way to update their profile information or manage their account settings. Users expect to change their name, email, password, and view their subscription/usage.

## Users
- All authenticated users who need to update their profile
- Users who want to change their password for security
- Users who want to check their usage limits (free tier vs paid)

## User Stories
- As a user, I want to change my display name
- As a user, I want to update my email address
- As a user, I want to change my password
- As a user, I want to see how much content I've processed this month
- As a user, I want to delete my account and all my data

## Proposed Solution
Add user settings page with profile management:

**Settings Page** (`/settings`):
- Navigation in header: "Settings" link
- Sections:
  1. **Profile**: Name, email (read-only for MVP)
  2. **Security**: Change password (current password + new password)
  3. **Usage**: Show usage stats (content processed this month, limits)
  4. **Account**: Delete account with confirmation

**Profile Section**:
- Input field for name (editable)
- Email display (read-only, can add email change later)
- Save button

**Security Section**:
- Current password input
- New password input with strength indicator
- Confirm new password input
- Update password button
- Error handling for incorrect current password

**Usage Section**:
- Display content processed this month
- Display limit (e.g., "1 / 10" for free tier)
- Progress bar or visual indicator
- Link to upgrade (placeholder for billing)

**Account Section**:
- "Delete Account" button (red/warning style)
- Confirmation modal with warning text
- Type confirmation: "Type DELETE to confirm"
- Cascade delete: removes user, all content, all outputs

## Scope
### In Scope
- Update user name
- Change password (verify current password)
- Display usage statistics (content count this month)
- Delete account with confirmation
- Settings page navigation

### Out of Scope
- Email change (requires verification flow)
- 2FA/MFA setup
- Subscription management (billing)
- API key management
- OAuth providers (Google, etc.)

## Technical Approach
- **Frontend**: New page `/settings/page.tsx` with sections
- **API**: PUT `/api/user` for name update
- **API**: POST `/api/user/password` for password change
- **API**: DELETE `/api/user` for account deletion
- **DB**: Usage model already exists, query for stats
- **State**: Form validation with Zod
- **Testing**: Verify all settings actions work
