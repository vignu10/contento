# PRD: Dashboard Search and Filter

## Problem
As users process more content, their content library grows. Without search and filter functionality, finding specific content becomes difficult. Users expect to be able to search by title, filter by status, and sort by date.

## Users
- Power users with 50+ content items
- Users looking for specific content to update or share
- Users who want to see only completed/failed content
- All users as their content library grows

## User Stories
- As a user, I want to search my content by title so I can find specific items quickly
- As a user, I want to filter by status (pending, processing, completed, failed)
- As a user, I want to filter by source type (YouTube, file upload)
- As a user, I want to sort by date (newest first, oldest first)
- As a user, I want to clear filters to see all content again

## Proposed Solution
Add search and filter controls to the dashboard content list:

**Search Bar:**
- Input field above content list
- Placeholder: "Search by title..."
- Real-time filtering as user types (debounced)
- Case-insensitive search

**Filter Controls:**
- Status dropdown: All, Pending, Processing, Completed, Failed
- Source type dropdown: All, YouTube, File Upload
- Date sort dropdown: Newest, Oldest
- "Clear Filters" button (visible when filters active)

**Combined Filtering:**
- Search + filters work together
- Clear filters resets to "All" view
- Show count of filtered results

**Empty State for Filtered Results:**
- "No content matches your filters"
- "Clear filters to see all content" button

## Scope
### In Scope
- Search by title (real-time, case-insensitive)
- Filter by status (All, Pending, Processing, Completed, Failed)
- Filter by source type (All, YouTube, File Upload)
- Sort by date (Newest, Oldest)
- Clear filters button
- Combined search + filter functionality
- Result count display
- Empty state for no results

### Out of Scope
- Advanced search (date range, transcript search)
- Saved search/filter presets
- Export filtered results (can use export all for now)
- Filter by output type
- Grouping by date/month

## Technical Approach
- **Frontend**: Search input, filter dropdowns, state management for filters
- **API**: GET `/api/content?search=...&status=...&sourceType=...&sort=...`
- **DB**: Prisma queries with where/orderBy
- **Debounce**: 300ms delay on search input to reduce API calls
- **State**: React state for active filters
- **URL**: Optional: persist filters in URL params for shareability

## Acceptance Criteria
- [ ] Search bar appears above content list
- [ ] Typing in search filters content by title in real-time
- [ ] Status dropdown filters by selected status
- [ ] Source type dropdown filters by selected source type
- [ ] Sort dropdown changes order of content
- [ ] Filters work together (search + status + source)
- [ ] Clear filters button resets all filters
- [ ] Result count shows how many items match filters
- [ ] Empty state displays when no results match
- [ ] Search is debounced (300ms)
- [ ] URL params reflect current filters (optional)

## Design Notes
- Place filters in toolbar above content list
- Use consistent spacing and alignment
- Show active filter indicators (e.g., "Status: Completed (3)")
- Keyboard support: Tab through filters, Enter to submit
- Mobile: Stack filters vertically on small screens
