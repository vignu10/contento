# PRD: Content Export & Download

## Problem
Users can copy content to clipboard, but there's no way to download files or export multiple formats at once. Users want to save all outputs for offline use, archiving, or posting later.

## Users
- Content creators who want to save outputs for later posting
- Agencies who need to deliver outputs to clients as files
- Users who want to keep a local archive of their repurposed content

## User Stories
- As a user, I want to download each output as a file (TXT, MD, JSON)
- As a user, I want to download all outputs at once as a ZIP file
- As a user, I want to download the transcript as a TXT file
- As a user, I want to export content to CSV for analysis

## Proposed Solution
Add export and download capabilities:

**Individual Downloads**:
- Each output card has a "Download" button
- Download options: TXT, Markdown, JSON
- Twitter thread → each tweet as separate file or single file
- LinkedIn post → text file
- Newsletter → Markdown or HTML
- TikTok clips → JSON with timestamps and scripts
- Quotes → JSON or plain text
- SEO summary → Markdown
- Instagram caption → text file

**Bulk Export**:
- "Export All" button on content detail page
- Downloads a ZIP file containing all outputs
- Organized by format (twitter/, linkedin/, newsletter/, etc.)
- Include transcript in root of ZIP

**Dashboard Export**:
- "Export History" button on dashboard
- Downloads CSV with all content (title, status, date, sourceType)
- For analytics and record-keeping

## Scope
### In Scope
- Download individual outputs as TXT, MD, or JSON
- Bulk export all outputs as ZIP
- Download transcript as TXT
- Export content history as CSV from dashboard
- Proper filename generation (e.g., "contento-twitter-2024-03-09.txt")

### Out of Scope
- Schedule export (email outputs automatically)
- Export to Google Drive, Dropbox, etc.
- PDF generation (separate feature)
- Quote graphic image generation (separate feature)

## Technical Approach
- **Frontend**: Download button on each output, Export All button
- **API**: New GET endpoint `/api/content/[id]/export?format=zip|json|csv`
- **Backend**: Use JSZip for ZIP creation, create CSV strings
- **File generation**: Format outputs appropriately for each download type
- **Browser**: Use `<a>` with `download` attribute or Blob URL
- **Testing**: Verify downloads work, ZIP contains correct files

## Acceptance Criteria
