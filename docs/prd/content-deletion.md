# PRD: Content Deletion

## Problem
Users have no way to delete content items they no longer need. Over time, the content library will grow, and users need the ability to remove old, test, or irrelevant content. This is basic content management functionality.

## Users
- All authenticated users who need to manage their content library
- Users who accidentally uploaded wrong content
- Users who want to clean up test content
- Users who have reached storage limits (future feature)

## User Stories
- As a user, I want to delete a content item from my dashboard
- As a user, I want to confirm before deleting to prevent accidental deletion
- As a user, I want to be able to delete multiple items at once (bulk delete)
- As a user, I want to see what outputs will be deleted along with the content

## Proposed Solution
Add delete functionality to the dashboard:

**Single Item Delete:**
- Add delete button (trash icon) to each content card in dashboard
- Clicking delete shows confirmation modal with:
  - "Are you sure you want to delete this content?"
  - "This will permanently delete the content, transcript, and all generated outputs."
  - Cancel and Delete buttons
- On delete: Remove content item, cascade delete all outputs
- Show loading state during delete
- Refresh list after successful delete

**Bulk Delete (optional for MVP):**
- Add checkboxes to content cards
- "Select All" button
- Bulk delete button (visible when items selected)
- Confirm before bulk delete

## Scope
### In Scope
- Delete single content item from dashboard
- Confirmation modal before delete
- Cascade delete (content → outputs → files from storage)
- Loading state during delete operation
- Success/error feedback (toast or inline message)
- Delete button in content detail page as well

### Out of Scope
- Bulk delete (can be added in Phase 2)
- Archive/soft delete (hard delete only for MVP)
- Undo delete (can be added later)
- Delete confirmation requires typing "DELETE" (simple confirm is enough for MVP)

## Technical Approach
- **Frontend**: Delete button in dashboard page.tsx, confirmation modal
- **API**: DELETE `/api/content/[id]` route
- **DB**: Prisma cascade delete configured (already in schema)
- **Storage**: Delete uploaded files from S3/local storage
- **State**: Optimistic UI update (remove item immediately, rollback on error)
- **Testing**: Verify cascade delete works, confirm button behavior

## Acceptance Criteria
- [ ] Delete button appears on each content card in dashboard
- [ ] Clicking delete shows confirmation modal
- [ ] Modal explains what will be deleted
- [ ] Canceling modal does nothing
- [ ] Confirming delete removes content from UI immediately (optimistic)
- [ ] Delete button shows loading state during API call
- [ ] Error is shown if delete fails
- [ ] Deleted content no longer appears after refresh
- [ ] Database cascade deletes all associated outputs
- [ ] Storage files are also deleted (if configured)
- [ ] Delete button also exists in content detail page
- [ ] Tests verify delete functionality

## Design Notes
- Use trash icon for delete button
- Modal should use warning colors (red border, red delete button)
- Show loading spinner during delete
- Success toast: "Content deleted successfully"
- Error toast: "Failed to delete content. Please try again."
